# 简历批改校准日志

最后更新：2026-04-07

## 目的

记录老师在真实测评多个简历时发现的误判、漏判和交互问题，并将其同步沉淀到：

- `src/data/reviewConfig.ts`
- `src/data/promptPlaybook.ts`
- `src/prompts/reviewPrompt.ts`
- `src/prompts/optimizePrompt.ts`
- `src/lib/openrouter.ts`
- `src/lib/markdown.ts`
- `src/components/MarkdownPreview.tsx`

## 当前校准主线

1. 在线简历，不是一页纸 PDF 简历。
2. 工作经历、项目经历按单段/单项目逐个批改，不能只看模块平均水平。
3. 非 `100` 分时，必须保留至少一个可执行问题，不能出现“四个模块全绿”。
4. 左侧高亮优先级：
   - 字段级
   - 段落/项目级
   - 模块级兜底
5. 右侧问题要去重，避免同类提醒重复出现。

## 已沉淀问题

### 2026-04-07

- 个人优势分点不足未稳定识别。
  - 处理：收紧为分点写法至少 8 点，并加入程序兜底。

- 工作经历和项目经历内容偏薄时，模型容易放过。
  - 处理：把“在线简历内容厚度不足”改成硬规则，并加入分数上限。

- 工作经历、项目经历被按模块整体判定，导致单段/单项目不达标被掩盖。
  - 处理：改成逐段工作经历、逐个项目检查；程序兜底会补出针对单段/单项目的 issue。

- 非满分时仍可能返回“无待调整项”。
  - 处理：要求批改 Agent 在非 `100` 分时必须输出至少一条可执行 issue；程序层再次兜底。

- 左侧高亮与右侧问题对应关系不稳定。
  - 处理：增加段落/项目级高亮，抽象问题不再强制打成零散片段。

- hover 卡片位置不合理，最后一段容易超出页面。
  - 处理：改为跟随光标，优先显示在光标上方。

- 个人优势、工作经历等问题存在重复提醒。
  - 处理：增加 issue 去重和泛化问题裁剪逻辑。

- 工作经历、项目经历的具体问题仍然高亮到整个模块，hover 区域不够精准。
  - 处理：高亮优先命中具体 title/字段；只有找不到更具体锚点时才退回段落级或模块级。

- 右侧给了具体段落/项目问题，但左侧没有准确对应到单段工作经历或单个项目。
  - 处理：工作经历、项目经历的 issue 必须尽量输出具体段落/具体项目标题；程序兜底也会补具体标题和精确 evidence。

- 发现新的硬要求：
  - 禁止概念解释类括号，例如“大模型（LLM）”
  - 禁止英文双引号包裹关键词，例如"工作流体系"
  - 禁止“擅长/熟练掌握/精通某个模型”类表达，例如“熟练掌握GPT-4o”
  - 处理：已同步写入 review rules、prompt calibration 和程序兜底检测。

- 工作经历展开不足类问题，应优先高亮对应公司/职位/时间这一行，而不是整段经历或整个模块。
  - 处理：工作经历的“展开不足/内容偏薄/缺背景缺结果”类 issue 统一锚定到对应 entry header。

- 项目经历问题应优先高亮具体项目 title 或对应字段（如职责描述/项目结果），不能整块染红整个项目模块。
  - 处理：项目 issue 按问题类型分别锚定 title、职责字段、背景字段或结果字段；generic issue 默认回落到项目 title。

- entry 级问题不应再默认用整段粉底提示，否则会让用户误以为整段都是错误点。
  - 处理：移除 entry 默认底色，改为精确字段高亮优先；只有 active/fallback 时保留轻量轮廓提示。
